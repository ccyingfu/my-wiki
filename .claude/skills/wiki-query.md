# Wiki Query — 知识查询

> 从已有的结构化知识中检索、综合、生成答案。

---

## 前置检查

1. **确认路径**：`/Users/liyingfu/Desktop/my-wiki`
2. **读取索引**：必须先读 `wiki/index.md`（总入口），再定位子索引
3. **读取 Schema**：如需了解页面格式约定，读 `wiki/schema.md`

**❌ 不要去 `raw/` 翻原始资料** — Query 只查 Wiki 层。除非 Wiki 完全没有相关信息且用户明确要求深度搜索。

---

## 执行流程

### Step 1：理解问题 & 定位页面

```
1. 分析用户问题的关键意图词和实体
2. 读 wiki/index.md → 找到相关分类
3. 读对应的 _index-*.md → 列出具体目标页面
4. 输出「待阅读页面清单」
```

**问题→索引映射表**：

| 问题类型 | 先查 | 再查 |
|---------|-----|------|
| "XX股票怎么样？" | `_index-stocks.md` | 个股页 + 关联板块页 |
| "光伏板块趋势？" | `_index-sectors.md` | 板块页 + 相关个股页 + 指标页 |
| "XX分析方法？" | `_index-concepts.md` | frameworks/ 下对应页 |
| "持仓整体情况？" | `_index-stocks.md`(持仓) | `_index-analyses.md`(最近复盘) |
| "A vs B？" | 两个实体页 | 共同关联板块/概念页 |

### Step 2：深入阅读

```
逐一阅读 Step 1 定位的所有页面：
- 记录每页与问题的相关性（高/中/低）
- 提取关键信息及引用位置
- 标注信息的时间新鲜度
```

### Step 3：选择输出形式 & 综合作答

#### 形式 A：直接回答（简单问答）

```markdown
【答案】

**依据：**
- [{页面名}]({相对路径})："{引用内容}"
- [{页面名}]({相对路径})："{引用内容}"

**置信度**：高/中/低（基于覆盖的资料量）
```

#### 形式 B：对比表格（比较类）

```markdown
# {A} vs {B} — 对比分析

| 维度 | {A} | {B} | 评价 |
|-----|-----|-----|------|
| PE(TTM) | xx | xx | ... |
| ROE | xx% | xx% | ... |
| 核心逻辑 | ... | ... | ... |
| 风险点 | ... | ... | ... |

## 结论
{综合判断}
**来源**：[A页](path/A.md) · [B页](path/B.md)
```

#### 形式 C：完整报告（写回 Wiki，有长期价值）

```markdown
---
title: "{标题}"
type: analysis / comparison / deep-dive
created: {日期}
related-stocks: [...]
related-sectors: [...]
sources: wiki (综合 N 页)
tags: [...]
---

# {标题}

## 执行摘要
{一段话结论}

## 详细分析
{分章节展开}

## 结论与建议
{可操作的}

## 来源索引
| 信息点 | 来自页面 |
|-------|---------|
| ... | [...]({...}.md) |
```

保存路径：
- 对比 → `wiki/analyses/comparisons/{A}-vs-{B}.md`
- 深度专题 → `wiki/analyses/deep-dives/{主题}-{日期}.md`

#### 形式 D：Marp 幻灯片（演示用）

```markdown
---
marp: true
theme: gaia
paginate: true
---

# {标题}

---

## 第一部分
...

---

## 总结
...
```

### Step 4：留存决策（关键）

每次 Query 结束时评估：

| 条件 | 动作 |
|-----|------|
| 一次性问答，无复用价值 | 不存 |
| 有新发现/新综合结论 | **询问是否写回** |
| 用户说"存下来" | 写入 + 更新索引+日志 |
| 生成了表格/报告 | 自动写入 `analyses/` |

**留存操作序列**：
1. 写文件到 `wiki/analyses/` 子目录
2. 更新对应的 `_index-*.md`
3. 更新 `wiki/index.md` 计数
4. 追加 `wiki/log.md`:
```markdown
## [{YYYY-MM-DD}] query | {问题摘要} → 产出: {文件路径}
- **问题**：{原始问题}
- **涉及阅读**：{N 个页面列表}
- **产出**：{保存的文件}
- **关键发现**：{一句话}
```
5. Git commit

### Step 5：汇报结果

```
📖 查询完成

❓ 问题：{原始问题}
📚 阅读了 N 个页面：{列表}
✅ 回答：（摘要）

📎 输出：直接回答 / 表格 / 报告(已存: path) / 幻灯片
💾 留存状态：未留存 / 已写入Wiki / 待确认

💡 延伸建议：
  - 可能相关的其他问题
  - 建议补充的资料方向
  - 发现的信息空白
```

## 高级用法

### 跨维度查询
"从基本面、技术面、资金面三个角度分析我的持仓"
→ 分维度定位 → 分别作答 → 综合判断

### 时间线查询
"澜起科技的认知演变？"
→ 读个股页「投资逻辑演进」章节 → 按时间线呈现

### 矛盾排查查询
"Wiki 里有哪些矛盾？"
→ 直接读 `meta/contradictions.md` → 逐条展示

### 空白发现查询
"AI算力板块还缺什么信息？"
→ 读板块页+相关个股页 → 对比研究框架 → 列出缺失维度

---

## 注意事项

1. **必须引用来源** — 每个论断注明来自哪个页面
2. **区分"有"和"没有"** — Wiki 没有的信息诚实说明，不编造
3. **标注置信度** — 低置信度要提示谨慎参考
4. **好的查询 = 新知识** — 积极建议留存有价值的结果
5. **Query 默认只读** — 不修改已有页面（除非留存时写新文件）
