## 一、HBM 是什么？

**HBM（High Bandwidth Memory，高带宽内存）** 是一种新型的 3D 堆叠 DRAM 技术，专门为需要**极高数据吞吐量**的场景设计——尤其是 AI/GPU 计算。

### 核心原理：解决"内存墙"问题

AI 大模型的计算瓶颈不在算力，而在"喂数据"的速度。GPU 计算速度极快但传统 DDR 给数据速度远跟不上，GPU 大部分时间在"等数据"，算力严重浪费。HBM 就是通过 3D 堆叠把数据传输通道从"小窗口"升级成"高速公路"。

### 技术原理：3D 堆叠 + TSV

**关键技术：**

| 技术 | 作用 | 意义 |
|------|------|------|
| **TSV（硅通孔）** | 垂直穿透芯片的微型导线 | 数据不用绕路，直接上下传输 |
| **3D 堆叠** | 8-12层DRAM芯片叠在一起 | 省面积+短距离=高带宽 |
| **硅中介层** | 连接HBM和GPU的高速通道 | 1024+条数据通道同时传输 |

### 数据对比：HBM vs DDR5

| 参数 | DDR5-6400 | HBM3E | 差距 |
|------|-----------|-------|------|
| **带宽** | ~51 GB/s（单条） | **~1.2 TB/s**（单颗） | **~24倍** |
| **位宽** | 64 bit | **1024 bit** | 16倍 |
| **功耗** | 较高 | **低30%+** | - |
| **封装体积** | 大（PCB占用多） | **极小**（紧贴GPU） | - |
| **适用场景** | 通用计算/服务器 | **AI训练/推理、HPC** | - |

| **终端应用** | 英伟达、AMD、英特尔、Google | AI加速器、GPU、TPU |

### HBM 在 AI 算力链中的位置

```
绿电(阳光电源) → 供电 → IDC(润泽) → 运行
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
         AI芯片(GPU)     光通信(新易盛)   内存互连(澜起)
              │               │               │
              ▼               ▼               ▼
         ┌─────────┐    ┌──────────┐    ┌──────────┐
         │ HBM内存  │    │800G光模块 │    │DDR5/CXL  │
         │ (本主题)  │    │(集群通信) │    │(CPU内存) │
         └─────────┘    └──────────┘    └──────────┘
              │
    GPU内部"超短距离"      光模块解决"远距离"     澜起解决
    的数据喂给问题          GPU之间的通信问题      CPU-GPU之间的互连问题
```

**关键认知：HBM 和澜起科技做的内存接口芯片是不同生态位，互补非竞争。**
- **HBM**：GPU核心↔显存，超短距离，片内
- **澜起 DDR5 RCD**：CPU↔主内存，中等距离，主板
- **澜起 CXL**：CPU↔扩展内存/加速器，远距离，机柜级
- **新易盛光模块**：GPU↔GPU跨节点，超远距离，数据中心级

## 六、与持仓关联分析

### 澜起科技 ↔ HBM（互补非竞争）

澜起不直接参与HBM，但CXL技术可间接与HBM协同。未来CXL"内存池化"可能让多台服务器共享HBM资源。

### 中芯国际 ↔ HBM

短期不能代工HBM（需先进DRAM制程+TSV），但国产替代大趋势下长鑫存储/长江存储有望突破，5年内国产HBM难有规模。

### A股 HBM 相关标的

**直接标的（当前未持仓）：**
- 长电科技/通富微电：HBM先进封装（TSV/CoWoS替代）
- 雅克科技/华海诚科：HBM材料（环氧树脂/Underfill）
- 联瑞新材/壹石通：HBM用球形硅微粉

**间接标的（已持仓）：**
- 澜起科技：内存接口芯片（DDR5/CXL），HBM的互补生态位

> 💡 A股没有HBM的直接龙头标的（DRAM三巨头都是韩/美企业），只有封装/材料/设备的间接标的。

> **HBM 是 AI 芯片的"燃料管"——AI 模型越大、参数越多，需要的显存带宽就越大。没有 HBM，再强的 GPU 也跑不动大模型。HBM 成本已占 AI GPU 总成本的 35%，是 AI 基础设施中比 GPU 芯片还贵的单一组件。**
